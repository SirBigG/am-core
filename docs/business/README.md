# Business Docs

Use this folder for product, domain, operations, content, and business-facing documentation.

Keep technical implementation details in `docs/engineering/`.

## Current Topics

- `domains/` - domain-driven notes for the business areas represented by the application.

## Domain Documentation Standard

Each domain note should prefer business language over code language. It should capture:

- Purpose and user value.
- Main actors.
- Core workflows.
- Business rules and invariants.
- Important states and lifecycle transitions.
- Integrations or neighboring domains.
- Open questions.

Domain notes do not need to be perfect before work starts. They should become more accurate every time a feature, bug fix, or investigation teaches us something.
