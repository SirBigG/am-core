# Documentation

This folder keeps project knowledge that should survive beyond a single task or chat.

## Structure

- `engineering/` - technical plans, architecture notes, investigations, upgrade reports, security notes, and implementation records.
- `business/` - product, domain, operations, content, and other business-facing documentation.

Prefer dated documents inside topic folders for investigations and plans, for example:

- `engineering/package-upgrades/YYYY-MM-DD-topic.md`
- `engineering/security/YYYY-MM-DD-topic.md`
- `business/YYYY-MM-DD-topic.md`
